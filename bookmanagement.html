Book Management System With Simple User Graphical Interface

Building a GUI to create and manage a book database using Tkinter and SQLite3

In this article, we are creating a simple tool for managing book details. This tool could be used to organize and maintain a personal book library. The main purpose of this article is to demonstrate the creation of a simple graphical user interface (GUI) with Tkinter library and to establish a backend connection to SQL databases in order to manage data and to reflect changes on the frontend.

Required libraries

  • SQLite3: used to interact with sql databases. It provides an SQL interface.
  • Tkinter:tkinter is the standard built-in GUI framework for Python. It provides an easy way to create GUI applications.

Architecture

In order to access information about the book list we are managing, our SQL database should contain details such as book title, author, published year and ISBN. (Another useful information could be our reading progress).

Our application should enable the user to access the database and interact with it using a series of functionalities that could be executed using SQL:

  • Viewing the book list
  • Searching for a book
  • Adding a new book to the collection
  • updating information
  • Deleting a book

Our goal here then is to create a connection to our database in the backend, implement our functionalities using mysql and connect each functionality to the right button in our frontend.

Front-end

Our GUI will contains the following tkinter elements:

Entries Text Boxes for title, author, year and isbn
Labels Each entry will be supported with a label
Buttons Each function will be executed by clicking on a certain button
Listbox Provides a view of our database elements
Scrollbar Allows navigation through the list

Back-end

The backend takes care of performing the SQL tasks triggered by the operations in the frontend. IT contains the following functions :

  1. Add Entry :adds a new row (a tuple) into the SQL table. This tuple consists of the entries given by the user : name, author, year and isbn. 
    (“INSERT INTO book VALUES (NULL,?,?,?, ?)”, (title, author,year, isbn,))
  2. View all :displays all the rows that are present in the table.
    ("SELECT * from book WHERE UPPER(title)=UPPER(?) OR UPPER(author)=UPPER(?) OR UPPER(year)=UPPER(?) or UPPER(isbn)=UPPER(?)",(title,author, year, isbn))
  3. Search entry :displays the rows given search entries typed by the user. 
    (“INSERT INTO book VALUES (NULL,?,?,?,?)”, (title, author,year, isbn,))

    *Using UPPER enables the user to perform case insensitive search.
